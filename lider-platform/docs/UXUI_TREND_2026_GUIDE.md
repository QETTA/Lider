## 🎯 1. 비전 및 핵심 가치 (Business Alignment)

### 1.1 LIDER 플랫폼 UX 비전
- **AI-First 업무 오케스트레이션**: 사용자가 AI를 "사용"하는 것이 아니라 "함께 일"하는 경험
- **Zero-Friction Automation**: 클릭 한 번으로 복잡한 업무 프로세스 완료
- **Trust-by-Design**: 모든 AI 출력물에 대한 투명한 근거 제공

### 1.2 2026년 핵심 UX KPI
| KPI | 목표값 | 측정 방법 |
|-----|--------|----------|
| 작업 완료 시간 단축 | 70%↓ | TTV (Time-to-Value) |
| AI 응답 신뢰도 | 95%+ | Grounded Answer Rate |
| 사용자 재방문률 | 80%+ | DAU/MAU 비율 |
| 행동 미리보기 승인율 | 90%+ | Preview-to-Execution Rate |

## 🎨 2. 2026년 주요 UX/UI 트렌드 적용

### 2.1 신경 다양성 기반 디자인 (Neurodiverse UX)
**사업적 근거**: 직장인의 15-20%가 신경 발달 다양성을 가짐 (ADHD, 자폐 스펙트럼 등)

```
[적용 규칙]
✓ 인지 부하 최소화: 한 화면에 3개 이하의 주요 액션
✓ 집중 모드: 방해 요소 제거, 몰입형 인터페이스
✓ 예측 가능한 패턴: 일관된 레이아웃과 동작
✓ 자극 조절: 애니메이션 속도, 색상 대비 조절 가능
```

**LIDER 적용**:
- Document Extraction: 진행률을 명확히 표시, 예상 완료 시간 노출
- Action Preview: 리스크를 색상/아이콘으로 직관적 표현 (🔴고위험 🟡중위험 🟢저위험)
- Focus Mode: 사이드바 숨기기, 전체 화면 모드 지원

---

### 2.2 생성형 UI (Generative UI / Dynamic Interface)
**사업적 근거**: 정적 UI → 사용자 행동/선호에 따라 실시간 변화하는 UI

```
[적용 규칙]
✓ 컨텍스트 기반 레이아웃: 작업 유형에 따라 UI 재구성
✓ 개인화된 툴바: 자주 사용하는 기능 자동 노출
✓ 적응형 정보 밀도: 사용자 숙련도에 따른 상세도 조절
```

**LIDER 적용**:
- Assist 페이지: 사용자 역할(관리자/일반)에 따라 추천 툴 변경
- Extract 결과: 신뢰도에 따라 확장/축소 가능한 필드 표시
- Action Preview: 액션 유형에 따라 미리보기 카드 자동 생성

---

### 2.3 투명한 AI (Transparent AI / Explainable UI)
**사업적 근거**: AI 블랙박스에 대한 사용자 불신 해소, 규제 대응 (EU AI Act 등)

```
[적용 규칙]
✓ 출처 표시: 모든 AI 응답의 근거 데이터 출시
✓ 확신도 시각화: AI의 확신 정도를 그래픽으로 표현
✓ 대안 제시: AI가 고려한 다른 옵션 노출
✓ 수정 가능성: AI 출력 직접 수정/피드백 UI
```

**LIDER 적용**:
- 모든 응답 하단에 "근거 보기" 버튼 (인용 출처, 툴 호출 기록)
- Confidence Score: 추출 필드별 신뢰도 게이지 표시
- "이 응답이 도움이 되었나요?" 피드백 메커니즘
- Action Preview: "왜 이 액션을 추천하나요?" 설명 패널

---

### 2.4 멀티모달 인터랙션 (Multimodal Interaction)
**사업적 근거**: 업무 환경에서 음성/터치/키보드 상황별 최적 입력 방식 필요

```
[적용 규칙]
✓ 음성 명령: "티켓 #12345를 해결로 표시"
✓ 드래그앤드롭: 문서 업로드, 필드 매핑
✓ 키보드 단축키: 전력 유저를 위한 단축키 체계
✓ 터치 최적화: 태블릿/모바일 대응
```

**LIDER 적용**:
- Voice-to-Action: 모바일 앱에서 음성으로 액션 트리거
- 파일 업로드: 드래그앤드롭 영역 명확히 표시
- 단축키 안내: Ctrl+K (커맨드 팔레트), / (빠른 검색)

---

### 2.5 감정 인식형 피드백 (Emotion-Aware Feedback)
**사업적 근거**: AI 실패 시 사용자의 답답함/불만을 감지하고 대응

```
[적용 규칙]
✓ 사용자 반응 모니터링: 재시도, 취소, 비정상적 패턴 감지
✓ 적응형 에러 메시지: 기술적 에러 → 사용자 친화적 설명
✓ 회복 경험: 실패 시 자동 대안 제시
```

**LIDER 적용**:
- 추출 실패 시: "텍스트가 흐릿한가요? 다시 촬영하거나 파일을 업로드해보세요"
- 액션 실패 시: 롤백 옵션 자동 노출, 고객센터 연결 버튼
- Rate Limit: 남은 시간과 대안(캐시된 결과) 안내

---

### 2.6 지속 가능한 디자인 (Sustainable/Minimal Digital Design)
**사업적 근거**: ESG 경영, 탄소 중립 목표, 디지털 탈진(Digital Fatigue) 방지

```
[적용 규칙]
✓ 다크 모드 기본: OLED 대응, 에너지 절감
✓ 데이터 효율: 불필요한 API 호출/이미지 최소화
✓ 접근성 우선: WCAG 2.2 AA 준수
✓ 디지털 웰빙: 사용 시간 알림, 휴식 권유
```

**LIDER 적용**:
- 다크 모드 시스템 연동 (OS 설정 자동 감지)
- 이미지 최적화: WebP, 지연 로딩, 썸네일
- 접근성: 스크린 리더 대응, 고대비 모드, 폰트 크기 조절
- 디지털 웰빙: 30분 사용 시 "잠시 휴식을 취하세요" 팝업

## 🏗️ 3. 디자인 시스템 아키텍처

### 3.1 토큰 기반 디자인 (Design Tokens)
```yaml
# colors.yml
primary:
  50: "#EFF6FF"   # Lightest - 배경
  100: "#DBEAFE"
  500: "#3B82F6"  # Primary - 브랜드색
  600: "#2563EB"  # Hover
  900: "#1E3A8A"  # Darkest - 텍스트

semantic:
  success: "#22C55E"
  warning: "#EAB308"
  error: "#EF4444"
  info: "#3B82F6"
  risk-high: "#DC2626"
  risk-medium: "#F59E0B"
  risk-low: "#10B981"

# typography.yml
font-family:
  sans: "'Pretendard', 'Noto Sans KR', sans-serif"
  mono: "'JetBrains Mono', 'D2Coding', monospace"

font-size:
  xs: "12px"   # 보조 정보
  sm: "14px"   # 기본 텍스트
  base: "16px" # 본문
  lg: "18px"   # 강조
  xl: "24px"   # 제목
  2xl: "32px"  # 페이지 타이틀

# spacing.yml (8px 기반)
space:
  1: "4px"
  2: "8px"
  4: "16px"
  6: "24px"
  8: "32px"
  12: "48px"
```

## 🎭 5. 모션 및 마이크로 인터랙션

### 5.1 2026 모션 가이드라인
```
✓ 의미 있는 모션: 상태 변화, 피드백, 방향성 표시
✓ 성능 최적화: 60fps, will-change 속성 활용
✓ 접근성 고려: prefers-reduced-motion 지원
✓ 브랜드 정체성: 부드러운 스프링 애니메이션 (0.3s, ease-out)
```

### 5.2 LIDER 특화 애니메이션
| 상황 | 애니메이션 | 목적 |
|------|----------|------|
| AI 응답 생성 | 타이핑 인디케이터 | 시스템 작동 중 표시 |
| 추출 완료 | 필드별 페이드인 | 진행 상황 시각화 |
| 리스크 감지 | 진동 + 색상 변화 | 주의 집중 유도 |
| 액션 성공 | 체크마크 + 축소 | 긍정적 피드백 |
| 롤백 가능 | 되감기 애니메이션 | 실행 취소 가능성 표시 |

## 🔐 6. 보안 및 프라이버시 UX

### 6.1 PII 처리 UI 가이드
```
[시각적 처리]
✓ 마스킹 표시: [EMAIL], [PHONE], [주민번호]
✓ 접근 권한 요청: "고객 연락처를 확인하려면 인증이 필요합니다"
✓ 감사 로그: "이 데이터 접근이 기록되었습니다"
```

### 6.2 액션 승인 UX
```
3단계 확인 프로세스:
1. Preview 생성 → 자동 분석 결과 표시
2. 사용자 검토 → 리스크/영향 확인
3. 최종 승인 → "이 작업은 되돌릴 수 없습니다" 강조
```
