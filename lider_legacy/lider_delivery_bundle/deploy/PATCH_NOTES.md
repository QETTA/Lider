# Lider 최종 디자인 패치 노트

## 상태
- 빌드 버전: `v0.2-patched`
- 형태: 정적 배포본
- 목적: 최종 디자인 검토 + 배포 프리뷰

## 전제
원본 최종 디자인 소스가 제공되지 않아, 이번 패치는 **현재 채팅에서 드러난 요구사항과 일반적인 최종 디자인 오류 패턴**을 기준으로 재구성되었습니다.

## 반영한 패치 항목

### 1. AI 응답과 확정 데이터의 구분 강화
기존에 자주 생기는 문제는 AI가 만든 설명문과 시스템이 보장한 사실이 같은 시각 계층으로 섞이는 것입니다.  
이번 패치에서는 아래처럼 분리했습니다.

- `검증된 사실` 영역 분리
- `추천 / 요약` 영역 분리
- `출처` 배지 노출
- `불확실성` 항목 별도 표기

### 2. 위험 액션의 직접 실행 UI 차단
`승인 / 변경 / 종료 / 전송` 같은 액션이 AI 결과와 바로 연결되면 사고가 납니다.  
이번 패치에서는 다음을 고정했습니다.

- 액션은 `Preview` 단계 먼저
- 권한 / 정책 / 확인 절차가 시각적으로 먼저 노출
- `Execute` 버튼은 데모상 비활성 처리
- `Missing checks`를 화면 중앙에 노출

### 3. 문서 추출 플로우 분리
문서 업로드 화면에서 자주 생기는 오류는 “추출 결과”와 “최종 자연어 설명”이 한 번에 섞여서 수정이 어렵다는 점입니다.  
이번 패치에서는 다음처럼 분리했습니다.

- 업로드/샘플 선택
- 구조화 JSON 추출
- Validator 경고
- 사람용 요약

### 4. 검색 결과의 근거 부족 문제 보완
통합 검색/어시스트 화면에서 추정 답변이 단정적으로 보이는 문제가 흔합니다.  
이번 패치에서는 다음을 반영했습니다.

- 검색 플랜 표시
- 툴 체인 표시
- 근거 facts 카드
- source chip 표시
- 확인 필요 문구 고정

### 5. 모바일/좁은 폭 레이아웃 보정
최종 디자인에서 흔한 깨짐 포인트를 정리했습니다.

- 카드 grid 자동 축소
- 긴 코드/JSON 줄바꿈
- sticky header 과도한 높이 축소
- 탭 overflow 대응
- CTA 그룹 세로 정렬 지원

### 6. 접근성 보정
- 대비가 낮은 상태 배지 개선
- 키보드 focus ring 추가
- hover 이외에도 상태 구분 가능하게 처리
- 정보 카드 제목/본문 간 위계 보정

### 7. 정보 구조 재정렬
이번 패치 배포본은 아래 5개 화면 축으로 정리했습니다.

- Overview
- Search Assist
- Document Extract
- Action Preview
- Router Map

### 8. 배포 편의성 개선
- 빌드 없는 정적 배포
- `index.html` 단순 진입
- CSS/JS 분리
- 외부 CDN 의존성 제거

## 이번 패치가 해결하는 핵심 문제
- “AI가 말했으니 실행된다”처럼 보이는 위험한 UX
- 근거 없는 답변의 과신 유도
- 문서 추출과 요약의 혼합
- 긴 JSON / 긴 텍스트 overflow
- 모바일 레이아웃 깨짐
- 정보 우선순위 혼선

## 다음 단계 권장
원본 디자인 소스가 들어오면 아래 순서로 실제 diff 기반 2차 패치를 하는 것이 좋습니다.

1. 원본 디자인 컴포넌트 맵핑
2. 시안 대비 차이점 목록화
3. 컴포넌트 단위 patch diff
4. 반응형/접근성 회귀 테스트
5. 실제 API 더미 연결
